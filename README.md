# 8-Puzzle Interactive Game

Web implementation of the classic 8-puzzle game with artificial intelligence. Play against another human or challenge an agent — all executable in the browser, no backend installation required.

## Characteristics

-  **Two game modes**: Human vs. Human and Human vs. AI
-  **AI with two algorithms**: A* with out-of-place tiles, A* with Manhattan distance
-  **Three difficulty levels** with different goal states
-  **Real-time statistics**: timer, moves, nodes explored
-  **Guaranteed generation of solvable instances** (shuffle from the goal)
-  **Pure frontend**: HTML + CSS + vanilla JavaScript, no frameworks or backend

## Installation and Use

### Prerequisites

- [Node.js](https://nodejs.org/) installed (includes `npx`)
- Modern browser (Chrome, Firefox, Edge)

### Steps

**1. Clone or download the repository**

```bash
git clone https://github.com/DSebas09/8-puzzle-web.git
cd 8-puzzle
```

**2. Start the development server**

```bash
npx live-server
```

> `live-server` is downloaded automatically with `npx`; no global installation is required.

**3. Open your browser**

The application will be available at:

```
http://127.0.0.1:8080
```

> The server automatically reloads the browser every time you save a file.

## Game Modes

### Human vs. Human

Two boards appear on the screen with the **same initial instance**. Each player solves their own board by clicking on the tiles adjacent to the empty space. **The first to reach the goal state wins**.

### Human vs. AI

One board for the human player and one for the AI agent. The agent solves the puzzle automatically using the algorithm corresponding to the selected difficulty level. The AI's moves are **visualized step by step** for educational purposes.

## Difficulty Levels

| Level      | Goal State            | Visual Pattern         | AI Used                | Initial Shuffle |
|------------|-----------------------|------------------------|------------------------|-----------------|
| **Easy**   | `[1,2,3,4,5,6,7,8,0]` | Natural Reading (→)    | A* misplaced tiles     | ~20 moves       |
| **Normal** | `[1,2,3,8,0,4,7,6,5]` | Spiral                 | A* Manhattan distance  | ~50 moves       |
| **Hard**   | `[1,4,7,2,5,8,3,6,0]` | Transposed (columns ↓) | A* Manhattan optimized | ~100 moves      |

The initial states are generated by **shuffle from the goal state**, ensuring that every generated puzzle is solvable without needing to check inversion parity.
